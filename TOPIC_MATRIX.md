## 함정 포인트 (출제 시 반드시 다뤄야 할 변별 항목)

1. NULL 비교에 `=`/`<>` 사용 불가, 오직 `IS NULL`/`IS NOT NULL`
2. `COUNT(*)`만 NULL 포함, 나머지 집계함수는 NULL 무시
3. `NVL2(a,b,c)` 인수 구조: a가 NULL이 아니면 b, NULL이면 c (`NVL`과 다름)
4. Oracle은 NULL을 최댓값으로 정렬, SQL Server는 최솟값으로
5. `DENSE_RANK`는 중간 순위 없음, `RANK`는 건너뜀, `ROW_NUMBER`는 동일값도 고유
6. `ROLLUP(A,B)`는 순서 의존, `CUBE`/`GROUPING SETS`는 순서 무관
7. `TRUNCATE`는 UNDO 미생성 (DELETE보다 빠름), ROLLBACK 불가
8. Oracle 수동 COMMIT, SQL Server AUTO COMMIT, DDL은 양쪽 모두 AUTO
9. 식별자관계 = 자식 PK에 부모 PK 포함 (조인 최소화), 비식별자관계 = 자식 일반 속성 (조인 증가)
10. ROWNUM은 `>` 비교 불가, `<=` 또는 `=1`만 가능
11. NATURAL JOIN은 ALIAS/테이블명 명시 불가, 동일 이름/타입/도메인 자동 조인
12. `USING` 절은 컬럼명 같아야 가능, ALIAS 명시 불가
13. CTAS(CREATE TABLE AS SELECT)는 NOT NULL만 복제, 다른 제약조건 미복제
14. PK 칼럼 순서: 앞쪽이 `=` 조건이어야 인덱스 효율 (범위 조건은 뒤로)
